# 게임 관리 CLI

이 문서는 게임 API와 상호작용하기 위한 명령줄 인터페이스인 `game-cli.ts` 스크립트 사용법을 안내합니다. 이 도구는 개발자의 테스트 및 관리 목적으로 사용됩니다.

## 과제 요구사항 및 `game-cli.ts`의 충족

본 과제의 주요 모듈인 "Basic Pong을 서버 측 Pong으로 교체 및 API 구현"에서는 CLI를 통한 게임의 부분적인 사용을 명확히 요구하고 있습니다. 원문 요구사항은 다음과 같습니다:

> Create an API that exposes the necessary resources and endpoints to interact with the Pong game, allowing **partial usage of the game via the Command-Line Interface (CLI)** and web interface.

여기서 핵심은 **"CLI를 통한 게임의 부분적인 사용(partial usage)"**의 의미를 정확히 파악하는 것입니다. 물론, 정보를 얻기 위해 데이터를 읽는 '조회(read-only)' 기능도 '사용'의 한 형태로 볼 수 있습니다. 하지만 이 요구사항은 단순히 데이터를 읽는 것을 넘어, 게임의 상태에 직접적인 영향을 미치는 **상호작용(interaction)**을 포함하는 것으로 해석해야 과제의 의도에 더욱 부합합니다. 게임을 생성하고 취소하는 것은 게임의 생명주기에 직접 관여하는 능동적인 행위인 반면, 게임 히스토리 조회는 이미 종료된 게임의 기록을 보는 수동적인 기능에 가깝기 때문입니다.

이러한 **'부분적인 사용'을 '상호작용'으로 해석하고 능동적인 기능을 포함해야 하는 이유**는 다음과 같습니다:

1.  **'사용(Usage)' 단어의 의미와 API의 본질**: 요구사항에서 '조회(viewing)'나 '읽기(reading)'가 아닌 '사용(usage)'이라는 단어를 선택한 것은 중요합니다. '사용한다'는 것은 일반적으로 어떤 대상의 기능을 활용하여 원하는 목적을 이루는 능동적인 행위를 의미합니다. 또한, API(Application Programming Interface)의 'Interface'는 두 시스템이 상호작용하는 접점이며, API를 '사용한다'는 것은 그 접점을 통해 데이터를 요청(Read)하고, 새로운 데이터를 생성(Create)하거나, 기존 데이터를 수정(Update) 또는 삭제(Delete)하는 모든 행위(CRUD)를 포괄하는 개념으로 보는 것이 자연스럽습니다. 여기서 '조회'는 CRUD 중 Read에 해당하지만, 과제의 핵심 목표를 달성하기 위해서는 Create, Update, Delete와 같은 능동적 상호작용이 강조되어야 합니다.
2.  **요구사항의 목적: API의 독립성과 완전성 증명**: 이 과제의 핵심 목표 중 하나는 프론트엔드(웹 인터페이스)와 백엔드(API)를 분리하여, 백엔드 API가 독립적으로 완결된 기능을 수행할 수 있음을 보여주는 것입니다. CLI를 통해 게임을 `create`하고 `cancel`할 수 있다는 것을 보여줌으로써, 웹 UI가 없어도 API만으로 게임의 전체 생명주기(life cycle)를 관리할 수 있다는 사실을 강력하게 증명할 수 있습니다. 이는 API가 단순한 데이터 제공자를 넘어, 하나의 완전한 서비스로서 기능한다는 것을 보여주는 가장 확실한 방법입니다.
3.  **'부분적(Partial)'의 의미 해석**: 여기서 '부분적'이라는 말은 '전체(Full)'의 반대 개념입니다. '전체 사용'이 "웹 인터페이스를 통해 실시간으로 그래픽을 보며 게임을 플레이하는 것"이라면, '부분적인 사용'은 그 외의 핵심 기능들을 사용하는 것을 의미합니다. 게임을 생성하고, 상태를 확인하고, 취소하는 것은 실시간 플레이는 아니지만 게임을 관리하는 핵심 기능의 일부입니다. 따라서 이러한 관리 기능을 CLI로 제공하는 것이 '부분적인 사용'이라는 요구사항에 정확히 부합합니다.

`game-cli.ts`는 다음 기능을 제공하여 이러한 요구사항을 충족합니다.

- **사용자 인증**: `login` 명령어를 통해 CLI에서 웹 애플리케이션에 로그인하고, API 사용을 위한 인증 토큰을 획득 및 저장합니다.
- **게임 관리 (생성)**: `create` 명령어를 통해 새로운 게임 매치를 생성하여, 서버의 게임 생성 API가 정상적으로 동작함을 입증합니다.
- **게임 상태 조회**: `status` 명령어를 통해 특정 게임의 현재 상태를 API로부터 조회하고 출력합니다. 이는 실시간 게임 플레이가 아닌, 특정 시점의 게임 상태를 '조회'하는 관리 도구의 역할을 수행합니다.
- **게임 취소**: `cancel` 명령어를 통해 진행 중인 게임을 취소하여, 게임 관리 API의 핵심 기능을 보여줍니다.

이러한 기능들은 CLI가 완전한 게임 클라이언트가 아니라, API를 사용하는 '관리 도구' 또는 '테스트 도구'로서의 역할을 명확히 합니다. 특히, CLI를 통해 게임을 만들고 취소할 수 있다는 점은 백엔드 API가 특정 프론트엔드에 종속되지 않고 유연하게 사용될 수 있음을 증명하며, 이는 API의 독립성을 보여주는 중요한 지표입니다. 따라서 `create` 및 `cancel`과 같은 기능의 구현은 과제 요구사항을 충실히 만족시키기 위한 필수적인 요소입니다.

## 전제 조건

CLI를 사용하기 전에 `ts-node`가 설치되어 있고 백엔드 서버가 실행 중인지 확인하세요.

## 위치

스크립트는 `backend/scripts/game-cli.ts`에 있습니다.

## 명령어

CLI는 다음 명령어를 지원합니다:

- `login`
- `create`
- `status`
- `cancel`

모든 명령어는 `make game-cli ARGS='...'` 형식을 사용하여 실행해야 합니다. 셸(zsh 등)의 예기치 않은 확장을 방지하기 위해 전체 인수를 작은따옴표로 묶는 것이 가장 안전합니다.

### `login`

사용자를 인증하고 현재 디렉토리에 `.clitoken` 파일로 액세스 토큰을 저장합니다. 이 토큰은 이후 모든 명령에 사용됩니다.

**사용법:**

```bash
make game-cli ARGS='login --email "<귀하의_이메일>" --password "<귀하의_비밀번호>"'
```

**인수 (Arguments):**

- `--email`: 등록된 이메일 주소.
- `--password`: 비밀번호.

**예시:**

```bash
make game-cli ARGS='login --email "user@example.com" --password "password123!@#"'
```

### `create`

새로운 게임을 생성합니다.

**사용법:**

```bash
make game-cli ARGS='create <유형> [옵션]'
```

**인수 (Arguments):**

- `유형`: 생성할 게임의 유형입니다. `local` 또는 `ai`가 될 수 있습니다.

**옵션 (Options):**

- `--opponent <이름>`: (`local` 게임에 필수) 게스트 상대방의 표시 이름입니다.
- `--difficulty <레벨>`: (`ai` 게임에 선택 사항) AI의 난이도입니다. `easy`, `medium`, `hard`가 될 수 있습니다.

**예시:**

- 로컬 1대1 게임 생성:
  ```bash
  make game-cli ARGS='create local --opponent "Player2"'
  ```
- 중간 난이도 AI와의 게임 생성:
  ```bash
  make game-cli ARGS='create ai --difficulty medium'
  ```

### `status`

특정 게임의 상태를 조회합니다.

**사용법:**

```bash
make game-cli ARGS='status "<게임ID>"'
```

**인수 (Arguments):**

- `게임ID`: 확인할 게임의 ID입니다.

**예시:**

```bash
make game-cli ARGS='status "some-game-uuid"'
```

### `cancel`

진행 중인 게임을 취소합니다.

**사용법:**

```bash
make game-cli ARGS='cancel "<게임ID>"'
```

**인수 (Arguments):**

- `게임ID`: 취소할 게임의 ID입니다.

**예시:**

```bash
make game-cli ARGS='cancel "some-game-uuid"'
```

## 중요 고려사항: 도구의 목적과 책임

이 CLI 도구는 실제 사용자를 위한 서비스가 아닌, **백엔드 API의 독립성을 검증하기 위한 개발 및 테스트용 유틸리티**입니다. 따라서 이 도구의 사용에는 몇 가지 중요한 전제와 책임이 따릅니다.

1.  **명시적인 리소스 정리 책임**: CLI를 통해 `create` 명령으로 생성된 게임 세션은, 사용자가 웹소켓으로 접속하기 전까지 `waiting` 상태로 서버 메모리에 유지됩니다. 만약 `cancel` 명령어를 통해 명시적으로 정리되지 않으면, 이 세션은 시작되지 않은 채 '고아 세션(Orphan Session)'으로 남아 잠재적인 메모리 누수 지점이 될 수 있습니다. 따라서 이 CLI를 사용하는 개발자는 **테스트 완료 후 반드시 `cancel` 명령어를 실행하여 생성된 게임 세션을 정리할 책임**이 있습니다.

2.  **제한된 사용 시나리오**: 이 도구는 `login` -> `create` -> `status` -> `cancel`로 이어지는 **고정된 시나리오** 하에서 API의 기능(게임 생명주기 관리)을 독립적으로 테스트하기 위해 만들어졌습니다. 평가 및 테스트 과정에서도 이 순서에 따른 단일 실행을 전제로 합니다.

3.  **웹 인터페이스와의 차이점**: 실제 서비스 환경에서는 웹 인터페이스를 통해 게임이 관리됩니다. 웹 환경에서는 사용자의 페이지 이탈이나 비정상적인 연결 종료 시에도 활성 웹소켓 연결 상태를 감지하여 서버가 자동으로 게임을 중단 처리하고 리소스를 정리하는 메커니즘이 구현되어 있습니다. 따라서 본 CLI 도구의 동작 방식이 실제 서비스 안정성에 영향을 미치지는 않습니다.
